## 1. 项目目标

青择校是一个面向陕西职业教育择校场景的微信小程序。产品只解决四件事：搜索、筛选、列表、详情。它不是招生平台，不做营销推荐，不做社区，不做个人中心，不做信息流。

核心目标是帮助初中毕业生、高中毕业生和家长用最少步骤了解一所职业学校的真实性、基础情况、专业方向、费用区间和 AI 客观解读。

## 2. 项目边界

允许功能：

- 学校搜索
- 专业搜索
- 条件筛选
- 学校列表
- 学校详情
- 首次手机号授权
- 学校详情页打分入口

禁止功能：

- 收藏
- 个人中心
- 推荐流
- 发现页
- 社区
- 动态
- 招生咨询营销入口
- 底部 TabBar
- 传统招生广告模块

## 3. 产品架构

页面数量保持极少，降低用户认知负担。

```text
Splash 加载授权页
  -> Home 首页搜索
      -> Schools 搜索结果列表
          -> SchoolDetail 学校详情
```

全局不配置底部导航。所有页面保留微信胶囊区域和顶部安全留白，不显示传统标题栏。

## 4. 工程架构

项目采用页面、业务模块、基础组件、服务层、类型层分离的结构。

```text
src/
  app.config.ts
  app.scss
  app.ts
  assets/
    icons/
    images/
    logo/
  components/
    base/
    business/
    feedback/
  constants/
    design.ts
    filters.ts
    routes.ts
  features/
    auth/
    home/
    school-detail/
    school-search/
  hooks/
  mock/
  pages/
    splash/
    home/
    schools/
    school-detail/
  services/
  styles/
    tokens.scss
    mixins.scss
    global.scss
  types/
  utils/
```

分层职责：

- `pages`：页面入口，只做页面组合和路由参数处理。
- `features`：业务模块，承载搜索、筛选、详情、授权等场景。
- `components/base`：基础 UI，如搜索框、标签、玻璃按钮。
- `components/business`：业务组件，如学校卡片、AI 报告、评分图。
- `services`：接口请求、mock 切换、数据适配。
- `types`：核心实体类型。
- `constants`：路由、筛选项、设计常量。
- `styles`：全局 SCSS token、mixin、通用样式。
- `utils`：格式化、参数序列化、评分换算、数据清洗。

## 5. 技术选型

框架：

- Taro 4：多端能力成熟，适合微信小程序工程化开发。
- React：组件化表达清晰，适合后续模块扩展。
- TypeScript：保证数据结构、接口字段、组件 props 可控。
- SCSS：适合沉淀设计 token、mixin 和玻璃拟态样式。

开发工具：

- Cursor：主要代码编辑和工程管理。
- 微信开发者工具：预览、真机调试、上传、体验版、发布。
- Figma 或 Stitch：承载设计稿、组件样式、视觉规范。

## 6. 页面技术方案

### 6.1 Splash 加载授权页

职责：

- 展示小程序 Logo。
- 检查本地授权状态。
- 首次进入触发微信手机号授权。
- 授权成功后进入首页。

关键点：

- 不提供个人中心。
- 授权只作为进入产品的安全门槛。
- 授权状态存储在本地和服务端会话中。

### 6.2 Home 首页

职责：

- 展示主标题“搜学校 搜专业”。
- 展示高识别度搜索框。
- 展示最多 5 个热词。

交互：

- 输入关键词后点击搜索进入列表页。
- 点击热词直接进入列表页。
- 页面不出现任何其他内容。

### 6.3 Schools 列表页

职责：

- 顶部复用搜索框。
- 展示筛选栏。
- 展示学校卡片列表。

筛选项：

- 地区
- 学费
- 专业
- 资质
- 类型

要求：

- 所有筛选可取消。
- 一屏至少展示 3 个学校。
- “查看详情”只能是文字入口，不做按钮。

### 6.4 SchoolDetail 详情页

职责：

- 展示学校图片组。
- 展示学校名称和标签。
- 重点展示 AI 解读报告。
- 展示关键数据、热门专业、多维评分。
- 底部展示液态玻璃“打分”按钮。

关键点：

- 图片区域参考抖音团购商家页，一行 2 到 3 张。
- AI 来源使用箭头入口，不直接堆文字。
- 多维评分采用 A+ 体系和五边形评分图。

## 7. 组件系统

基础组件：

- `SafeTop`：处理微信胶囊和顶部安全区。
- `LogoSplash`：加载页品牌展示。
- `SearchBox`：首页和列表页搜索。
- `HotTag`：热词标签。
- `FilterBar`：筛选入口。
- `FilterSheet`：筛选弹层。
- `GlassTag`：液态玻璃标签。
- `GlassActionBar`：底部玻璃操作区。
- `EmptyState`：空结果。
- `LoadingState`：加载态。

业务组件：

- `SchoolCard`：列表学校卡片。
- `SchoolSearchHeader`：列表页顶部搜索区。
- `SchoolHeroGallery`：详情图片组。
- `SchoolAIInsight`：AI 解读报告。
- `SchoolKeyMetrics`：关键数据。
- `SchoolMajorSection`：热门专业。
- `SchoolScoreSection`：多维评分。
- `RadarPentagon`：五边形评分图。
- `GradeBadge`：A+ 等级。

组件原则：

- 组件 props 必须强类型。
- 组件只接收已清洗数据，不在组件内拼接口字段。
- 业务组件不可直接请求接口。
- 视觉样式优先使用设计 token。

## 9. 数据来源与合规边界

允许采集范围：

- 陕西职业学校
- 中专
- 大专
- 职业教育
- 成人教育

禁止采集范围：

- K12
- 本科院校
- 非陕西学校
- 与择校无关的营销内容

数据来源优先级：

1. 学校官网和官方公众号公开信息
2. 教育主管部门公开信息
3. 中国教育在线等公开教育平台
4. 公开舆论信息，用于 AI 摘要辅助，不作为唯一事实来源

服务端后续应保留来源记录、采集时间和字段置信度。

## 10. API 设计

首版可以用本地 mock 数据驱动页面，接口层保持真实 API 形态。

```text
GET /api/schools
  keyword
  region
  tuition
  major
  qualification
  type

GET /api/schools/:id

POST /api/auth/phone

POST /api/schools/:id/rating
```

返回要求：

- 列表接口返回轻量字段。
- 详情接口返回完整字段。
- AI 报告来源只返回 source id 和入口，不在详情页直接堆来源说明。

## 11. 状态管理

首版不引入大型全局状态库。

状态策略：

- 授权状态：`hooks/useAuth` + 本地缓存。
- 搜索参数：页面 query + 局部 state。
- 筛选状态：`features/school-search` 内维护。
- 学校数据：services 请求后页面局部持有。

只有当后续跨页面状态明显变多时，才考虑引入 Zustand。

## 12. 性能策略

- 首页极轻，不加载列表和详情数据。
- 列表卡片图片使用压缩图。
- 详情图片横向懒加载。
- 组件避免复杂嵌套和过深层级。
- SCSS 避免过多阴影叠加。
- 不使用大体积 UI 框架。

## 14. 验收标准

产品验收：

- 首页只有搜索、热词和留白。
- 无底部导航。
- 无个人中心。
- 无招生营销话术。
- 列表页一屏至少 3 个学校。
- 详情页 AI 解读是视觉重点。

工程验收：

- TypeScript 无明显类型错误。
- 页面路径清晰。
- 组件职责单一。
- 数据结构可替换真实接口。
- 微信开发者工具可正常预览。

视觉验收：

- 白色背景为主。
- 青色主题明确。
- 阴影轻，卡片层级清楚。
- 标签和 AI 卡片有半透明液态玻璃感。
- 没有传统招生平台和电商风。
